# Contributing to HealMeal

Thank you for your interest in contributing! We want to keep the HealMeal codebase clean and high-quality.

## 🤝 Code of Conduct

Please be respectful and professional in all communications.

## 📝 Pull Request Process

1. **Fork the repo** and create your branch from `main`.
2. **Standardize**: Ensure your code follows the patterns in [ARCHITECTURE.md](./ARCHITECTURE.md).
3. **Lint**: Run `flutter analyze` and fix all warnings.
4. **Test**: Run `flutter test` to ensure no regressions.
5. **Document**: Update relevant `.md` files if you change functionality or structure.
6. **Submit**: Open a PR with a clear description of your changes.

## 🎨 Style Guide

- Use **Trailing Commas** everywhere for better formatting.
- Prefer **Arrow Functions** for one-liners.
- Always specify **Member Visibility** (public/private).
- Avoid `dynamic` types; use `Object?` or specific types.

## 💬 Communication

For bugs or feature requests, please open a GitHub Issue.
